# Virtualization basics (THM) - Overview - Notes

Before virtualization, only one application could run on a single server. This led to wasted resources and high costs for a company due to them needing to buy multiple machines, more electricity, and cooling for one application to run a database, email service, or an application.
This led to the creation of the hypervisor: a referee that managed and created virtual computers and allowed each one to act independently to enable an application to run on shared hardware for one physical computer.
